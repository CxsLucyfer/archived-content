この記事ではアドオンを発行するプロセスを一通り見ていきます。単に始めたい場合は、AMO の Submit a New Add-on ページに移ります。

始めるには、レビューポリシーとD開発者の承諾に精通しておきます。

次に、発行するアドオンを準備して、すべてのファイルを.zip, .xpi, .crx, .xml いずれかの拡張子の ZIP アーカイブに追加します。拡張機能をパッケージするを見てください、同梱するファイルの詳細は、拡張機能の仕組みを見てください..zip, .xpi, .crx, .xml.

The maximum file size accepted is 200 MB. If your add-on is larger than 200 MB, it will fail validation.

アドオンに読みにくいコードが含まれる場合、例えば minification(最小化) や obfuscation(難読化)で作成されたコードでは、ソースコードパッケージを作成します。この要求の詳細は、Source Code Submission.を見てください。

開発者アカウントを https://addons.mozilla.org (AMO) で作成します:

  • Firefox アカウントをお持ちの場合、それで AMO に接続します: AMO に移動して、ページヘッダーの中で Log in をクリックして、Firefox アカウントでログインします。
  • Firefox アカウントを持っていない場合: AMO に移動して、ページヘッダーの中で Register(登録)をクリックします。

Add-ons Developer Hub に移動して Submit Your First Add-on か Submit a New Add-on. をクリックすると、次の:ページに移動します:

次のどれかを選びます:

  • On this site, アドオンを AMO に載せる場合。
    Continue をクリックして AMO に掲載する のガイドに従いやり方を見ます。
  • On your own, アドオンを自分で配布して、AMO に掲載しない場合。
    Continue をクリックして自己配信ガイド に従いやり方を見ます。

AMO に掲載する

  1. アップロード版のページはこう表示します。
  2. Select a file をクリックして、アドオンを選択してアップロードします。
  3. バリデータがアドオンの問題を確認してページが更新されます。
  5. アドオンがすべてのチェックを通ったら、次のメッセージを受け取ります:

    warnings(警告)だけがあるというメッセージを受け取ることもあり、それは警告の場所を指してしますが、必須ではありません: 投稿を続けることができます。
    それ以外では、次のようなメッセージを受け取ります:

    このメッセージは失敗している場所を教えてくれます。続けることはできません。問題を突き止めてステップ 1 に戻ります。
  6. ソースコードの提供が必要な場合、Browse をクリックしてソースコードパッケージを指定してアップロードします。
  7. アドオンの互換プラットフォームを選びます。
  8. Continueをクリックします。
  9. Describe Add-on(アドオン説明) のページの表示です。

    下記を与えます:
    • Name: アドオンの名前。
    • Add-on URL: AMO でのアドオンの URL。URL はアドオン名に応じて自動的に割り振られますが、変更するには Edit をクリックします。 注記: URL はユニークでなければならず、選択した URL を他のアドオンが使っている場合は警告されて、別のURLをセットする必要があるでしょう。
    • Summary: アドオンの要約を役立つように、説明が十分なように書く。
    • This add-on is experimental: アドオンが実験的だったり一般的な使用の用意ができていない。アドオンは掲載されるが見られるのは制限される。アドオンが一般的な使用の用意ができた時にこのフラグを削除できる。
    • This add-on requires payment, non-free services or software, or additional hardware: アドオンが完全に動作するために、ユーザーに追加の支払いが必要である。
    • Select up to 2 Firefox categories for this add-on: アドオンを説明するカテゴリーを選ぶ。
    • Select up to 2 Firefox for Android categories for this add-on: アドオンを説明するカテゴリーを選ぶ。
    • Support email and website: ユーザーが質問や問題や賞賛を送る時に連絡できる電子メールアドレスとウェブサイト。
    • License: アドオンの適切なライセンスを選ぶ。各ライセンスの詳しくは Details をクリックします。
    • This add-on has a privacy policy: ユーザーの端末からデータを送信する場合、何が送られてどのように使われるのかを説明するプライバシーポリシーが必要です。このチェックボックスをチェックしてプライバシーポリシーを与えます。
    • Notes to Reviewer: AMO レビュアーを助ける情報、例えばダミーアカウントのログイン詳細や、ソースコード情報や、そうしたもの。
  10. Submit Version.をクリックします。
  11. バージョン送信ページはこう表示します。電子メールの確認も受け取ります。

アドオンが AMO に公開されて利用できるようになり、ユーザーが発見してインストールできます。注意点として、しかし、アドオンが更なるレビューを受けることもあり、その場合は後のレビューの結果の通知を受け取るでしょう。

When you're ready to release a new version of your add-on, update it by visiting the add-on's page on addons.mozilla.org and uploading the new version. You must update it on the add-on's page, so AMO recognizes it as an update to an existing add-on and not a new one.

アドオンを一覧の中で目立たせるにはこの tips 学習してください。

自己配信

If you want Firefox to handle updates to your add-on, remember to include the applications key in your manifest.json with the update_url attribute set to point to an update manifest file.

  1. The Upload Version page displays.
  2. Click Select a file, select and upload your add-on.
  3. The validator checks the add-on for issues and the page updates.
  4. If your add-on passes all the checks, you receive the following message:

    You may receive a message that you only have warnings, it's advisable to address them, but not mandatory: you can continue with the submission.
    Otherwise, you receive a message similar to this:

    The message informs you of what failed. You'll not be able to continue. Address the issues and return to step 1.
  5. If you need to provide source code, click Browse and locate and upload your source code package.
  6. Select the add-on's compatible platform(s).
  7. Click Sign Add-on.
  8. The Version Signed page displays. You'll also receive an email confirmation.
  9. Click Download to obtain a signed copy of your add-on.

You can now distribute your add-on. Note, however, that your add-on may still be subject to further review, if it is you'll receive notification of the outcome of the review later.
